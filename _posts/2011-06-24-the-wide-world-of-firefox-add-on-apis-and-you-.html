The Firefox Add-on SDK is now 1.0! It's got a set of APIs that allow you to build basic restart-less add-ons without needing to know anything but JavaScript.

But the core set of APIs is clearly not complete. There are no APIs for creating or searching bookmarks nor for building sidebars or menus. There's a whole host of things you can do in XUL add-ons that the SDK doesn't yet have nice and clean JavaScript APIs for.

And I hope it never will.

BECAUSE... I think that the best way to both keep the SDK slim-and-trim *and* massively grow the feature set is for add-on developers to publish and share their modules, similar to how has been done in the Node community with NPM.

The Add-on SDK team will always be pushing things forward - adding infrastructural improvements, support for new Firefox features, fixing bugs and improving resource efficiency and performance. And this work is often higher priority than adding the API that you might need for your add-on.

So if you write a module for your add-on, and you think others might be interested in using it, please list it on the Jetpack Modules page. Maybe eventually there'll be a cool package manager for these modules, or maybe NPM itself could be adapted to use the Add-on Builder's repository of library packages.

Creating a module is easy. It's a JavaScript file that uses the CommonJS format for defining public APIs.
